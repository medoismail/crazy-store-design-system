Issues Found

🔴 Critical Issues (5)
1. color/black/ 50% — has a rogue space. Rename to color/black/50%
2. item spacing/0 — value is 2.84e-14 (float rounding), not 0. Reset to exactly 0
3. item spacing/s — named "s" while all others use numbers. Rename to item spacing/16
4. greyscale/text/negative v1 — "negative v1" is unclear. Rename to greyscale/text/inverse
5. greyscale/text/negative v2 — "negative v2" is unclear. Rename to greyscale/text/highlight
⚠️ Warnings (4)
1. color/azure/84 (#D1D5DC) overlaps with color/grey/91 (#E5E7EB) — both used as borders. Pick one per context.
2. greyscale/surface/subtle resolves to same hex as greyscale/surface/default — they're visually identical
3. 12 floating-point spacings (34.27, 70.66, etc.) appear auto-extracted from layouts, not intentional tokens. Clean up or move to a "layout" collection.
4. Font size scale jumps from 18→36 — missing intermediate sizes for subheadings
✅ What works well (keep as-is)
• Color naming convention color/{hue}/{lightness} is clear and consistent
• Semantic collection structure (surface, border, text) is well-organized
• Primitive → Semantic aliasing pattern is correct
• Radius, stroke, height, width tokens are clean
• Font family and weight tokens are functional
• Line height scale is complete for current needs
