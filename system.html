Discrepancies Found

Conflict

stack-gap-sm value mismatch

tokens.json: 12 (aliases space-12)
tokens 4.json: 8 (aliases space-8)
Two files define different values for the same semantic token. One must be chosen as source of truth.

Conflict

stack-gap-md value mismatch

tokens.json: 16 (aliases space-16)
tokens 4.json: 12 (aliases space-12)
Same conflict pattern. Likely one file is an older version.

Duplicate

Neutral/100 and Neutral/200 are identical

Both resolve to #FDFCF9. This makes greyscale/surface/default and greyscale/surface/subtle visually indistinguishable.

Duplicate

greyscale/surface/subtle = greyscale/surface/default

Both resolve to #FDFCF9. Subtle should be visually distinct from default to provide meaningful hierarchy. Consider using Neutral/300 (#FBF9F4) for subtle.

Naming

color/black/ 50% has rogue space

Variable name contains a space before "50%". Should be color/black/50%. This can cause lookup failures in code.

Naming

item spacing/0 is not exactly 0 (MCP data)

Value is approximately 0 (~0) instead of exactly 0. This can cause sub-pixel rendering issues and unexpected gaps in layouts.

Naming

item spacing/s uses letter instead of number (MCP data)

All other item spacing variables use numeric names (0, 10, 12, 20, 40, 150) but "s" breaks the pattern. Value is 16. Maps to stack-gap-md (= space-12) in the _spacing system.

Duplicate System

Two parallel spacing systems exist

The _spacing primitive collection (space-0 through space-64) and the legacy item spacing/* variables (from MCP data) both define spacing values. This creates ambiguity about which system is canonical. They also use different scales: _spacing uses a 4px grid; the legacy collection includes 10 and 150 which are off-grid. Legacy values should map to: item spacing/10 → space-12, item spacing/12 → space-12, item spacing/s → stack-gap-md, item spacing/20 → space-20, item spacing/40 → space-40.

What's Missing

These are recommendations only. Each item is labeled NEW to indicate it does not currently exist in the system.

Font Weights

Only font/weight/regular (400) and font/weight/bold (700) exist. Medium (500) and SemiBold (600) are needed for subtitle emphasis and UI labels.

font/weight/medium font/weight/semibold

Font Sizes

Missing sizes for small text (captions, labels) and mid-range headings.

font/size/xs font/size/sm font/size/xl

Line Heights

Small and large line heights for tight UI and spacious content blocks.

line height/16 line height/32

Border Radius

Only r-4 (radius/button) and r-12 (radius/card) exist. A mid-size radius and a pill/circle radius are essential.

r-8 / radius/input radius/full

Hover State Colors

No semantic tokens for interactive hover states. Components need predictable hover colors.

greyscale/surface/hover

Focus State Colors

No focus ring or border-focus token for accessibility compliance.

greyscale/border/focus

Semantic Hover/Focus Colors

The semantic colour collection only covers default, subtle, and disabled states. Active, hover, and focus are missing across surface, border, and text groups.

greyscale/surface/active greyscale/border/active

Recommended Architecture

A 3-layer token architecture ensures consistency, maintainability, and theme-ability. Components should never reference primitives directly.

Layer 3 — Components

Component Tokens

Use semantic tokens only. Never reference primitives directly.

Button.background = greyscale/surface/default Card.padding = card-padding Input.border = greyscale/border/default
Layer 2 — Semantics

Purpose-Driven Aliases

Named by intent (surface, text, gap, padding). Alias to primitives. Enable theming.

greyscale/surface/default greyscale/text/title card-padding stack-gap-sm
Layer 1 — Primitives

Raw Values

Scale-based, context-free values. Colors by hue/step, spacing by size.

Neutral/200 = #FDFCF9 DarkMidnightBlue/500 = #003462 space-24 = 24 space-48 = 48

Key Rules

  • Components reference Layer 2 (semantics), never Layer 1 (primitives)
  • Semantics alias Layer 1 (primitives), never raw hex values
  • Primitives are the single source of truth for raw values
  • Theming works by swapping Layer 1 values (e.g., dark mode swaps Neutral scale)
  • Consolidate legacy item spacing/* (MCP data) into _spacing primitives to eliminate the duplicate system
